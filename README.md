# Destination Board 2020
シンプルな行先掲示板アプリです。



# コンセプト

日本企業にありがちな情報を、最低限表示できるめちゃめちゃ簡単な行先掲示板です。（なのでドキュメントなども基本的に日本語ベースです）

タブレットなんかで職場でちょろっと行先をデジタルサイネージ的に見せられたらいいなと。

動作前提としては、非管理者のローカルユーザで常時起動しておくサイネージマシンで使われることを想定してます。

需要あるかわからないですが、自分がちょうど欲しかったので作り始めました。

ホワイトボードでやってると手が汚れたり、面倒なんですよね。

## 組織情報の表示

- 利用する組織の情報を表示することができます。

- 組織によってフロアにいるチームが疎らだと思ったので一部はオプションです。

- 表示する情報は[組織と人の設定](#組織と人の設定)から編集できます。

  

## 組織に所属する人のリスト表示

- 任意の組織、人の名前表示ができます。

- 最小単位の組織内では任意の順番で表示ができます。

- 組織内での責任者をドットで表示できます。

- 表示する情報は[組織と人の設定](#組織と人の設定)から編集できます。

- 名前を押下している間、会社支給の携帯電話の内線番号を表示することができます。

  

## 勤務ステータスの入力・表示

- 当日の出退勤ステータスを入力・表示することができます。

- 出勤している場合は緑、未出勤または退勤した場合は赤のドットで表示します。

- 以下の場合、フレックス出勤の表示をします。

  - 設定した勤務時間の開始時刻から30分以上前に出勤した場合
  - 設定した勤務時間の開始時刻から30分以上後に出勤された場合
  - 閾値は設定で変更できます。

- 以下の場合、フレックス退勤の表示をします。

  - 設定した勤務時間の終了時刻より前に退勤された場合

- 出勤している場合は緑、未出勤または退勤した場合は赤のドットで表示します。

  

## 行先情報の入力・表示・通知

- 出勤後に離席する際、行先を一覧から選択し、その情報を保持して表示することができます。

- 行先情報は以下から選択します。
  - 事業所内の場所（エリア、建物、階、部屋、食堂）
  - 出張（他事業所、社外）
  - その他（食事、トイレ、たばこ、社外など）

- 行先は[行先の設定](#行先の設定)から任意に増やすことができます。

- 行先は予め設定されたショートカットから選ぶこともできます。

- 行先はよく使われる場所から選ぶこともできます。

- 離席理由をリストから選ぶことができます。（オプション）

  - 会議
  - 作業
  - 試験
  - 休憩（食事、トイレ、たばこなど）
  - その他

- 離席理由は[離席理由の設定](#離席理由の設定)から任意に増やすことができます。

- TODO: 将来的には予定表アプリ（O365など）からの取得ができるようにしたいです。

  

## 戻り予定時間の入力・表示

- 離席先から戻る時間を選択して入力・表示することができます。

- 選択できる時間は30分単位です。

- 選択できる時間は現在よりも後の時間から設定された最大値の間です。

  

## 残業時間の入力・表示

- 本日の申請済残業時間の表示

  

## 離席情報のクリア

- 席に戻ったときに離席情報をクリアして在席状態にできます。

  

## 翌日以降の予定表示

- 退勤時に翌日以降の行先設定ができます。翌日以降の予定は以下から選択・入力します。

  - 翌日のフレックス出勤予定時刻

  - 翌日休暇

  - 翌日出張が直行・直帰

  - 翌日出張＋戻り予定時間

  - 長期出張の終了日

  - 長期休みの終了日

  - その他（期間未確定の休職）

    

## 各種動作設定

- アプリの起動中、OSの設定を画面を常時表示にするか選べます。（Default: Enable）

- アプリの起動時、アプリを全画面表示にするか選べます。（Default: Enable）
  
- F11でいつでも全画面表示とウィンドウ表示をトグルで切り替えが可能です。
  
- 退勤ルールなどに合わせて戻り予定時間の最大値を設定できます。（Default: 22:00）

- 勤務時間の設定ができます。（Default: 9:00-17:00）

- フレックス出勤の閾値が分で設定できます。（Default: 30）

- 以下の組織名称を表示をするか選べます。（Default: Disenable）

  - 会社名、本部、部
  - 課、係（チーム）は必ず表示されます。
  - 上記に限らず、直接紐づく人が表示対象の場合は必ず表示されます。
